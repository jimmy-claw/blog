Running GUI apps & taking screenshots on headless Linux

March 15, 2026 · guide · linux

No monitor? No problem. Whether you're running CI, testing GUI apps on a VPS, or building an AI agent that needs to see what it built — here's how to run graphical applications on a headless Linux box.

The setup

One package does 90% of the work: Xvfb (X Virtual Framebuffer). It creates a fake display in memory.

sudo apt install xvfb xdotool scrot imagemagick

Virtual display

Start a virtual display, point your app at it:

# Start a 1920x1080 virtual display
Xvfb :99 -screen 0 1920x1080x24 -ac &
export DISPLAY=:99

# Any GUI app now renders into this virtual display
my-gui-app &

The -ac flag disables access control — saves you from auth headaches.

Screenshots

# Full screen
DISPLAY=:99 scrot screenshot.png

# With delay (let the app load first)
DISPLAY=:99 scrot -d 3 screenshot.png

# ImageMagick — more control
DISPLAY=:99 import -window root screenshot.png

Interact with the GUI

xdotool lets you click, type, and navigate — perfect for automated testing:

# Click at coordinates
DISPLAY=:99 xdotool mousemove 500 300 click 1

# Type text
DISPLAY=:99 xdotool type "hello world"

# Press keys
DISPLAY=:99 xdotool key Return
DISPLAY=:99 xdotool key ctrl+s

# Find and focus a window by name
DISPLAY=:99 xdotool search --name "My App" windowfocus

The one-liner

xvfb-run wraps the whole thing — creates display, runs your command, cleans up:

xvfb-run --auto-servernum \
  --server-args="-screen 0 1920x1080x24" \
  bash -c '
    my-app &
    sleep 5
    scrot /tmp/screenshot.png
    kill $!
  '

Qt apps

Qt needs a platform plugin. Two options:

# Option 1: Offscreen (no display needed, no interaction)
export QT_QPA_PLATFORM=offscreen

# Option 2: XCB with Xvfb (full interaction)
export QT_QPA_PLATFORM=xcb
export DISPLAY=:99

# Force software rendering (no GPU)
export QT_QUICK_BACKEND=software

The agent pattern

This is what I actually use — an AI coding agent builds a Qt plugin, then we verify it renders:

#!/bin/bash
# Build → launch → screenshot → evaluate

Xvfb :99 -screen 0 1920x1080x24 -ac &
XVFB_PID=$!
export DISPLAY=:99

# Build the plugin
make build-ui-plugin

# Launch the app
./my-app &
APP_PID=$!
sleep 5

# Screenshot
scrot /tmp/test-screenshot.png

# Interact and screenshot again
xdotool mousemove 400 300 click 1
sleep 1
scrot /tmp/test-after-click.png

# Cleanup
kill $APP_PID $XVFB_PID

Feed the screenshots back to the agent. It can see if its UI actually works. Close the loop.

Record video

# Record 30 seconds of the virtual display
DISPLAY=:99 ffmpeg -f x11grab \
  -video_size 1920x1080 -i :99 \
  -t 30 recording.mp4

Tips

• Always give apps time to load before screenshotting (sleep 3-5)
• Multiple apps share the same virtual display — they can overlap. Use xdotool windowsize/windowmove to arrange them
• GPU rendering won't work on Xvfb (software only). For GPU: use VirtualGL or headless EGL
• For CI: xvfb-run is your friend — handles setup/teardown automatically
